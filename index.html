Notes

General things to remember:

  • html file (HyperText Markup Language) is the content displayed in the actual webpage
  • css file (Cascading Style Sheets) contains general information about the page styling
  • js file contains how interactions contained in your page will work

CSS things to remember:

  1. "RULES": are the ones that styles the html markup (head, body, h1, p, ul, ol, li, etc.)
  2. "CLASS SELECTOR": '.list' is e of them. Each HTML element that contains a class attribute set to list will get the styles that are defined within this selector.
  3. "ID SELECTOR": '#msg' is one of them. The HTML element that has its id attribute set to msg will get the styles that are defined within this selector.
  4. It uses inheritance mechanism from parent elements to their descendants (key aspects of CSS).
  5. To define a set of global CSS variables use a CSS rule with the ':root' selector.

JS thisngs to remember:

  1. "script" can be placed in head or right before the closing body. If the latter is choosed, then it gives the possibility to all the pages content to be displayed on the screen first, and then to load the script.
  2. "noscript" is a message that appears if JavaScript is deactivated, thus the code can detect and plan for when a feature isn't supported or available.
  3. Enabling 'strict mode' (in js) will let you know when you made mistakes, otherwise JavaScript will let you make mistakes handling them by itself, but it will be harder to find them in the future.
  4. Event Handler is a way to run a JavaScript function when the specified event occurs on the page. With 'document.querySelector' you can link the reference to the involved element. Before this command you find the variable with the reference stored.
  5. Console Message are hidden messages that won't appear on the webpage but that are displayed in the Developer Tools. These are extremely helpful to check your code.
